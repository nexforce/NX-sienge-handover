# Briefing Mestre, Curso Claude para Executivos

## Identidade do curso

**Nome de trabalho:** Claude para Executivos, do básico ao avançado
**Audiência:** Executivos e líderes (C-level, diretores, gerentes seniores) não-devs
**Duração total:** 8 horas (8 aulas de 60 min)
**Formato:** Híbrido (live síncrono + materiais de apoio + atividades práticas entre aulas)
**Pré-requisito:** Nenhum conhecimento técnico. Necessário apenas computador com permissão para instalar Claude Desktop / Cowork.

## Proposta de valor

Ao final do curso, o executivo:
1. Decide com clareza quando usar Claude vs alternativas (ChatGPT, Gemini, soluções internas)
2. Domina os 3 ambientes (Chat, Cowork, Code) e sabe quando delegar Code ao time técnico
3. Constrói ao menos 1 caso de uso real próprio (com seus dados, sua rotina)
4. Define plano de adoção do Claude para seu time (champion, casos prioritários, métricas, prazo)
5. Aplica governança e segurança desde o dia 1 (folders, backup, PII)

## Filosofia pedagógica

- **Aplicação > teoria.** Cada aula tem atividade prática.
- **Casos reais > exemplos genéricos.** Sempre que possível, usar contexto Nexforce (B2B tech LatAm, Marketplace + Services + Agents) ou contexto do próprio aluno.
- **ROI desde a aula 1.** Em vez de "aprenda esses conceitos", a primeira aula já mostra economia de tempo concreta.
- **Português.** Todo o conteúdo é entregue em PT-BR. Termos técnicos em inglês ficam em itálico na primeira menção.

## Lista de aulas

1. **Claude para executivos:** o que é, por que importa, diferenciais vs ChatGPT/Gemini, família de modelos (Opus, Sonnet, Haiku)
2. **Os 3 ambientes + pricing:** Chat, Cowork, Code (quando usar cada), pricing por user vs por org, limites e governança de consumo
3. **Claude Chat na prática:** interface, Projects, prompt engineering executivo, casos de uso diários
4. **Claude Cowork:** instalação, primeiro projeto, trabalhar com arquivos locais, demonstração ao vivo
5. **Connectors:** conectar Claude a Gmail, Slack, HubSpot, ClickUp, Drive, Notion, casos de uso reais
6. **CLAUDE.md, Memory, Skills, Subagents:** especializar Claude para tarefas e times específicos
7. **Plugins, Artifacts, Scheduled Actions:** dashboards vivos, automação recorrente, casos avançados
8. **Governança e adoção:** estrutura de folders, backup, PII, políticas, plano de adoção do piloto à escala

## Materiais de apoio por aula

Cada aula entrega ao aluno:
- Slide deck (15-25 slides)
- Cheat sheet (1 página) com comandos, prompts ou estruturas chave
- Atividade prática com critério de sucesso
- Lista de recursos externos (docs oficiais Anthropic, vídeos, exemplos)

## Avaliação

- **Atividade ao final de cada aula:** entrega prática vinculada à rotina do aluno
- **Projeto final (aula 8):** plano de adoção do Claude para o time do aluno
- **Certificado:** emitido após conclusão das 8 atividades + projeto final

## Casos Nexforce usados no curso

O curso usa o contexto Nexforce como base de exemplos (com permissão do Vitti):

- **Vitti (CEO)** usando Claude para conteúdo (LinkedIn, blog), análise de pipeline (HubSpot), preparação de calls, redação de propostas
- **Estrutura de folders do Vitti:** `/Users/vitti/Documents/Claude/Projects/[Area]/[projeto]/`
- **Projeto meta-agent:** exemplo de CLAUDE.md, skills, subagents bem estruturados
- **Unidades Nexforce:** Marketplace, Services, Agents (como diferentes times usam Claude de formas diferentes)
- **Metodologia OPMAX:** referência ao framework de gestão do Vitti

## Instruções para o agente de design

- **Tom:** sério, executivo, sem infantilização. Audiência sabe que isso vale dinheiro.
- **Visual:** densidade média. Não criar deck com slides de 1 palavra. Cada slide entrega informação acionável.
- **Sem em-dashes** (`—`). Usar vírgula, ponto ou reescrever.
- **Sem emojis** decorativos. Apenas como indicador visual (status, checkmark) quando agregam clareza.
- **Diagramas > texto corrido** sempre que possível.
- **Screenshots reais** da interface Claude (não mockups). Versão de referência: Claude Desktop / Cowork de maio 2026.

## Notas finais

- Cada aula tem briefing próprio (`01-...md` até `08-...md`) com: objetivo, breakdown de tempo, conteúdo detalhado, exemplos Nexforce, atividade prática, materiais de apoio.
- O design agent deve transformar cada briefing em deck + cheat sheet + roteiro de atividade.
- Dúvidas sobre escopo ou contexto: voltar ao Vitti antes de produzir.
